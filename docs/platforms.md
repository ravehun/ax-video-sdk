# 平台差异与能力边界(摘要)

本页用于快速理解同一套 API 在不同后端/芯片上的行为差异。详细语义以各模块文档为准。

## 后端

- `AX MSP`(板端): 图像内存通常位于 CMM/Pool，可直接喂给 VDEC/VENC/IVPS 等硬件模块。
- `AXCL`(算力卡): 设备侧图像默认在卡上，host 不可直接访问，需要时应显式 D2H。

## 常见能力边界(经验值)

### AX650

- H.264/H.265 decode/encode 常用能力齐全
- IVPS/OSD 能力较完整

### AX620E 系列(例如 AX630C/AX620Q/AX620QP)

- 具体编解码能力依赖芯片型号与驱动:
  - 部分平台可能仅支持 H.264 解码
  - 编码能力通常支持 H.264/H.265(以实际驱动为准)
- 多路能力通常不按高并发设计，现实目标以 `1~2 路 1080p` 为主

### AXCL

- 支持单进程多卡；建议一个 pipeline 固定绑定一张卡(`device_id`)
- PCIe 传输成本高:
  - 除非用户显式要求 host 访问，否则图像处理/编解码尽量在设备侧完成

## OSD

不同后端可绘制类型可能不同；当前版本通常只保证 AXCL 下 `rect` 稳定可用。
